# Assignment-Based DSL Syntax Design

## Goal

Change the KMapper DSL syntax from property-reference + infix style to natural assignment style while keeping all existing functionality.

## Syntax Change

```kotlin
// OLD
val userDto: UserDto = user.mapper {
    to::age map it.age.toString()
    to::name map "${it.firstName} ${it.lastName}"
    to::active.ignore()
}

// NEW
val userDto: UserDto = user.mapper {
    age = it.age.toString()
    name = "${it.firstName} ${it.lastName}"
    active.ignore()
}
```

The no-arg `user.mapper()` for identical classes remains unchanged.

## Architecture

### Runtime (`compiler-runtime`)

The `Mapper` class changes from `KProperty0`-based API to a class whose properties are synthetically generated by the FIR extension. The `it` parameter (source object) is passed as a constructor argument.

```kotlin
class Mapper<TO, FROM>(val it: FROM) {
    // Synthetic var properties generated per TO's constructor params
    // fun <T> T.ignore() recognized by compiler
}
```

### FIR Phase

Two extensions replace the current `KMapperFirMappingChecker`:

**A. `FirDeclarationGenerationExtension`** -- Generates synthetic `var` properties on `Mapper<TO, FROM>` matching `TO`'s primary constructor parameters. Provides IDE autocomplete and type checking.

**B. `FirVariableAssignmentChecker`** -- Validates mapping completeness:
- Intercepts assignments within the `mapper` lambda body
- Collects assigned properties and `.ignore()` calls
- Cross-references against `TO`'s constructor parameters (reuses `deepEqual` logic)
- Reports `MissingConstructorParameters` for unmapped fields without defaults

### IR Phase

`KMapperIrBuildMapperVisitor.createMapperImplementationFromCall()` changes mapping extraction:

- **Old:** Extract from `IrCall` statements (`map`/`ignore` infix calls) via `IrPropertyReference`
- **New:** Extract from `IrSetField`/`IrSetVariable` statements (property assignments) + `IrCall` for `ignore()`

Constructor building, nested mapping, enum mapping, list handling, and `it`-remapping logic remain unchanged.

## Testing

All existing integration tests updated to new syntax. Tests using `mapper()` with no lambda are unchanged. The `IntegrationTest` infrastructure stays the same -- only Kotlin source strings in test cases change.

## Files Changed

- `compiler-runtime/src/main/kotlin/community/flock/kmapper/Runtime.kt` -- New Mapper class
- `compiler-plugin/.../fir/KMapperFirMappingChecker.kt` -- Replace with FirVariableAssignmentChecker
- `compiler-plugin/.../KMapperFirExtensionRegistrar.kt` -- Register new FIR extensions
- New: `compiler-plugin/.../fir/KMapperFirDeclarationGenerator.kt` -- Synthetic property generation
- `compiler-plugin/.../ir/KMapperIrBuildMapperVisitor.kt` -- Read assignments instead of map calls
- `test-integration/.../KMapperTest.kt` -- Update all test syntax
- `README.md` -- Update examples
